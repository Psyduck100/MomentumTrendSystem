7. DEPLOY AS LIVE STRATEGY
   ────────────────────────
   The strategy is ready to trade, but follow these steps:
   
   a) Paper trade first
      - Use your broker's paper trading account
      - Verify that real fills match backtested entry/exit prices
      - Watch for slippage and execution issues
      - Run for at least 1 week
   
   b) Account for costs
      - Transaction costs: ~2-3 basis points per trade
      - Slippage: ~1 basis point typical for GLD
      - Taxes: May impact realized gains
      - Adjust backtest expectations accordingly
   
   c) Position sizing
      - Start with 25% of intended final size
      - Scale up gradually as you gain confidence
      - Monitor max drawdown in real-time
      - Have risk management rules
   
   d) Monitoring
      - Track daily MA values vs. latest price
      - Monitor for regime changes
      - Check for data quality issues
      - Keep detailed trading logs


8. UNDERSTAND THE ARCHITECTURE
   ────────────────────────────
   The refactored system uses design patterns:
   
   Strategy Pattern:
     - FallbackStrategy is abstract
     - CASH, TB3MS, IEF are concrete implementations
     - Add new fallbacks without changing backtest code
   
   Configuration Pattern:
     - All parameters in pmtl_config.py
     - Change tests by editing one line
     - Preset configs for common scenarios
   
   Dependency Injection:
     - Backtest engine accepts fallback as parameter
     - Zero coupling between components
     - Easy to test and modify
   
   Factory Pattern:
     - get_fallback_strategy() creates appropriate fallback
     - Decouples creation from usage
     - Easy to add new types
   
   See: PMTL_ARCHITECTURE.py for detailed explanation
